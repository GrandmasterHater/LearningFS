Решение и эталон расходятся по способу решения задач. В первом задании эталон предлагает рекурсивное объединение элементов с n до 1, для объединения используется оператор
сцепления списков "@" в результате получаем восходящий список от 1 до n. Во втором задании эталон аналогично первому заданию использует рекурсивное решение, когда элементы 
объединяются с n до 1, только теперь используется оператор "::" и значения добавляются в начало. В результате получается нисходящий список от n до 1. В третьем задании эталон
значения n от 1 до 3 обрабатывает в отдельных ветвях, для n < 1 возвращается пустой список, а остальные случаи обрабатываются выражением `evenn(n - 1) @ [2 * (n - 1)]` когда 
на каждом шаге рекурсии сцепляются списки из рекурсивного вызова `evenn(n - 1)` и значения в списке вычисленного по формуле `2 * (n - 1)`, которое вычисляет n-ый чётный элемент.

Решение предлагает иной подход. Во всех трёх задачах увидел общий принцип перебора, когда есть диапазон чисел и нужно идти в определённую сторону (по возрастанию/убыванию) с 
определённым шагом. Подготовил функцию итерирования в указанном диапазоне чисел с требуемым шагом. Для первой задачи функция использовалась с вызовом от 1 элемента до n с шагом 1.
Во второй задаче функция итерировала список в обратную сторону от n до 1 с шагом -1. В тертьей задаче использовал композицию функций - снача значение n попадало в функцию вычисления
стартового значение по формуле `n * 2 - 2` (по сути та же, что и в эталоне только наивная), а затем это передавалось функции итератору. Результирующая функция вызывалась для
значения n.
