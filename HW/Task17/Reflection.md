Эталон и решение используют разные алгоритмы решения задачи. Эталон определяет функцию subsets которая генерирует списки всех возможных множеств, затем через foldBack 
преобразует в список множеств, затем список множеств преобразует в множество множеств, затем фильтрует только те, где количество элементов равно k. 

Решение предлагает подход, когда определяем одну рекурсивную функцию generate_subsets которая срезу включает логику генерации и фильтрации множеств. Функция принимает массив чисел множества, 
ассумулятор текущего собираемого множества, и аккумулятор множеств. Сначала обрабатываем краевой случай, когда в исходном множестве на текущем уровне еще есть числа, но 
собираемое множество достигло нужного размера - в таком случае прерываем рекурсию и добавляем полученное множество в аккумулятор. Далее определена ветка отсечения для случаев,
когда в исходном списке осталось заведомо меньше чисел, чем нужно для требуемого подмножества - в таком случае возвращаем текущий аккумулятор. Ветка _, current_set, acc корректно
обрабатывает и слуйча когда массив пустой, но компилятор ругается, похоже when мешает компилятору определить покрытие случая когда массив пустой, для этого задал ветку `[], _, acc -> acc`.
Шаг рекурсии представляет из себя получение текущего элемента массива исходного массива и рекурсивные вызовы generate_subsets для случаев когда текущее число либо 
добавлено в массив либо нет. Затем вызываем generate_subsets для сгенерированного массива [1 .. n].
